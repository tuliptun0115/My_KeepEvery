# UI/UX Pro Max - 設計智能 (Design Intelligence)

針對網頁與行動應用程式的全面設計指南。包含 50 多種風格、161 個配色方案、57 個字體組合、161 種帶有推理規則的產品類型、99 條 UX 指南以及 10 個技術棧的 25 種圖表類型。可搜索的資料庫，提供基於優先級的建議。

## 何時適用

當任務涉及 **UI 結構、視覺設計決策、交互模式或使用者體驗品質控管** 時，應使用此技能。

### 必須使用 (Must Use)

在以下情況下，必須調用此技能：

- 設計新頁面（落地頁、儀表板、管理後台、SaaS、移動應用）
- 創建或重構 UI 組件（按鈕、模態框、表單、表格、圖表等）
- 選擇配色方案、字體系統、間距標準或佈局系統
- 審閱 UI 代碼的使用者體驗、無障礙性或視覺一致性
- 實作導航結構、動畫或響應式行為
- 做出產品級設計決策（風格、資訊層級、品牌表達）
- 提高介面的感知品質、清晰度或易用性

### 建議使用 (Recommended)

在以下情況下建議使用：

- UI 看起來「不夠專業」，但原因不明確時
- 收到關於易用性或體驗的修正建議時
- 上線前的 UI 品質優化
- 對齊跨平台設計 (Web / iOS / Android)
- 建立設計系統或可重用的組件庫

### 跳過 (Skip)

在以下情況下不需要使用：

- 純後端邏輯開發
- 僅涉及 API 或資料庫設計
- 與介面無關的效能優化
- 基礎設施或 DevOps 工作
- 非視覺化腳本或自動化任務

**決策標準**：如果任務會改變功能的 **外觀、感覺、移動或交互方式**，則應使用此技能。

## 優先級規則類別 (Priority Rules)

*供人類/AI 參考：按照優先級 1→10 決定首先關注哪個規則類別；需要時使用 `--domain <Domain>` 查詢詳情。*

| 優先級 | 類別 | 影響 | 領域 (Domain) | 關鍵檢查項目 (必備) | 反模式 (避免) |
|----------|----------|--------|--------|------------------------|------------------------|
| 1 | 無障礙 (Accessibility) | 關鍵 (CRITICAL) | `ux` | 對比度 4.5:1, Alt 文本, 鍵盤導航, Aria 標籤 | 移除焦點框, 無標籤的純圖標按鈕 |
| 2 | 觸控與交互 | 關鍵 (CRITICAL) | `ux` | 最小尺寸 44×44px, 8px+ 間距, 加載回饋 | 僅依賴懸停, 瞬時狀態切換 (0ms) |
| 3 | 效能 (Performance) | 高 (HIGH) | `ux` | WebP/AVIF, 懶加載, 預留空間 (CLS < 0.1) | 佈局抖動, 累計佈局位移 |
| 4 | 風格選擇 | 高 (HIGH) | `style`, `product` | 匹配產品類型, 一致性, SVG 圖標 (禁用 emoji) | 隨意混合扁平與擬物, 將 Emoji 當成功能圖標 |
| 5 | 佈局與響應式 | 高 (HIGH) | `ux` | 移動優先斷點, Viewport meta, 無水平滾動 | 水平滾動, 固定 px 容器寬度, 禁用縮放 |
| 6 | 排版與顏色 | 中 (MEDIUM) | `typography`, `color` | 基礎 16px, 行高 1.5, 語義化顏色 Token | 正文小於 12px, 灰底灰字, 組件中硬編碼 hex |
| 7 | 動畫 (Animation) | 中 (MEDIUM) | `ux` | 持續時間 150–300ms, 動作表達含義, 空間連續性 | 僅裝飾性動畫, 動畫化 width/height, 無減弱動態支援 |
| 8 | 表單與回饋 | 中 (MEDIUM) | `ux` | 可見標籤, 錯誤顯示在欄位旁, 漸進式揭露 | 僅 Placeholder 標籤, 錯誤僅顯示在頂部, 資訊過載 |
| 9 | 導航模式 | 高 (HIGH) | `ux` | 可預測的返回, 底部導航 ≤5, 深度連結 | 過載導航, 損壞的返回行為, 無深度連結 |
| 10 | 圖表與資料 | 低 (LOW) | `chart` | 圖例, 提示框, 無障礙色彩 | 僅依賴顏色來傳達資訊 |

## 快速參考指南 (Quick Reference)

### 1. 無障礙性 (Accessibility - 關鍵)

- `color-contrast`: 普通文本對比度至少 4.5:1 (大文本 3:1)。
- `focus-states`: 對交互元素顯示清晰的焦點框 (2–4px)。
- `alt-text`: 為有意義的圖片提供描述性 alt 文本。
- `aria-labels`: 為純圖標按鈕提供 aria-label；原生應用使用 accessibilityLabel。
- `heading-hierarchy`: 順序使用 h1→h6，不可跨級。
- `color-not-only`: 不要僅通過顏色傳達資訊（需添加圖標/文字）。
- `dynamic-type`: 支援系統字體縮放，避免文字增長時被截斷。
- `reduced-motion`: 尊重 prefers-reduced-motion 設置。

### 2. 觸控與交互 (Touch & Interaction - 關鍵)

- `touch-target-size`: 最小 44×44pt (Apple) / 48×48dp (Material)；必要時擴展視覺邊界外的點擊區域。
- `touch-spacing`: 觸控目標之間至少保持 8px/8dp 間距。
- `loading-buttons`: 在非同步操作期間禁用按鈕並顯示加載指示器。
- `press-feedback`: 按下時的視覺回饋（波紋/高亮；MD 狀態層）。
- `safe-area-awareness`: 確保主要觸控目標避開瀏海屏、動態島、手勢條和螢幕邊緣。

### 3. 效能與感知 (Performance - 高)

- `image-optimization`: 使用 WebP/AVIF，實作響應式圖片 (srcset)，延遲加載非關鍵資產。
- `content-jumping`: 為非同步內容預留空間以避免佈局跳動 (CLS)。
- `virtualize-lists`: 為 50 個以上的項目使用虛擬列表以提高滾動效能。
- `progressive-loading`: 對於超過 1 秒的操作，使用骨架屏/閃爍效果而非旋轉圖示。

### 4. 佈局與規範 (Layout & Spacing - 高)

- `mobile-first`: 先設計移動端，再擴展至平板與桌面。
- `spacing-scale`: 使用 4pt/8dp 增量間距系統。
- `8dp-grid`: 保持組件、部分和頁面層級的 4/8dp 間距節奏。
- `safe-area-compliance`: 標頭、底部欄與 CTA 欄必須遵守安全區域。

## 如何使用此技能

當使用者提出以下需求時，邀請此技能協助：

| 場景 | 觸發示例 | 從哪裡開始 |
|----------|-----------------|------------|
| **新項目 / 頁面** | "做一個 landing page"、"建立一個儀表板" | 步驟 1 → 步驟 2 (設計系統) |
| **新組件** | "創建一個價格卡片"、"添加一個模態框" | 步驟 3 (領域搜索: style, ux) |
| **選擇風格 / 顏色 / 字體** | "金融 App 適合什麼風格？"、"推薦配色" | 步驟 2 (設計系統) |
| **審閱現有 UI** | "檢查此頁面的 UX 問題"、"檢查無障礙性" | 上方的快速參考清單 |
| **修復 UI Bug** | "按鈕懸浮效果損壞"、"加載時佈局移位" | 快速參考 → 相關章節 |

### 實作流程：

#### 步驟 1：分析需求
提取產品類型 (SaaS, Tool, Media)、目標受眾、風格關鍵詞 (極簡, 活力) 與技術棧 (本專案為 Vite/React)。

#### 步驟 2：生成設計系統 (必要選項)
**一律先執行 `--design-system`** 以獲取帶有推理的完整建議：

```bash
python .agents/assets/ui-ux-pro-max/scripts/search.py "<產品類型> <行業> <關鍵詞>" --design-system [-p "項目名稱"]
```

#### 步驟 3：補充細節搜索 (視需求而定)
使用領域搜索獲取額外細節：

```bash
python .agents/assets/ui-ux-pro-max/scripts/search.py "<關鍵詞>" --domain <domain> [-n <最多結果數>]
```

**可用領域 (Domains):**
- `product`: 產品類型建議
- `style`: UI 風格、顏色效果
- `typography`: 字體配對建議
- `color`: 配色方案
- `ux`: 最佳實踐與反模式
- `chart`: 圖表類型建議
- `landing`: 頁面結構與 CTA 策略

#### 步驟 4：獲取技術棧指南 (React)
獲取 React 實作相關的最佳實踐：

```bash
python .agents/assets/ui-ux-pro-max/scripts/search.py "<關鍵詞>" --stack react
```

## 交付前檢查清單 (Pre-Delivery Checklist)

在交付 UI 代碼前，請確認以下項目：

### 視覺品質
- [ ] 不使用 Emoji 作為功能圖標（使用 SVG 代替）
- [ ] 所有圖標風格統一（來自同一家族）
- [ ] 按下狀態不會導致佈局位移或震動
- [ ] 使用語義化主題 Token（無硬編碼色值）

### 交互體驗
- [ ] 所有可點擊元素都有清晰的按下回饋
- [ ] 觸控目標符合最小尺寸要求 (>=44x44pt)
- [ ] 微交互時間保持在 150-300ms 區間
- [ ] 禁用狀態在視覺上清晰且無法交互

### 佈局與安全
- [ ] 遵守 Header、Tab Bar 與底部 CTA 欄的安全區域
- [ ] 滾動內容不會被固定欄位遮擋
- [ ] 已在小手機與平板（橫/豎屏）上驗證過
- [ ] 保持 4/8dp 的間距節奏
