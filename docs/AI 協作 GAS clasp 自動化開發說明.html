1 如何命令 AI 安裝與設定 clasp

當您的專案中有 Google Apps Script（`.js` 檔案）需要部署時，您可以在對話中直接把以下 Prompt 複製貼給您的 AI 助理：

💬 首次啟動與登入

告訴 AI 啟動登入機制。AI 會提供一個 Google 登入網址給您。

請在本地為我準備 GAS 程式碼。我想使用 clasp 來管理與自動部署。請幫我啟動 clasp login，並告訴我該如何授權。

💬 連接專案與推送

提供專案 ID 後，命令 AI 建立設定檔並一鍵推送到雲端。

這是我的 Apps Script 專案 ID：[貼上您的Script ID]。請建立 .clasp.json，並將 rootDir 設為專用資料夾以隔離檔案，最後執行 clasp push 將最新程式碼同步上雲端。

2 怎麼協作？雙向開發三步驟

一

開啟 Google 雲端 API 權限 (一生一次)

請先前往 Google Apps Script 使用者設定，將 Google Apps Script API 設為 「開啟 (ON)」。否則本地 clasp 會無法存取您的雲端專案。

二

完成瀏覽器授權登入

AI 在本地啟動 `clasp login` 後會產生一個瀏覽器授權網址。點選該網址登入您的 Google 帳號並允許存取。當網頁跳出 Logged in! 即可關閉。

三

提供 Script ID 供 AI 一鍵發布

在 Apps Script 網頁點選左側「專案設定 ⚙️」複製 Script ID 提供給 AI。AI 將會把程式碼自動打包推送，並安全更新您現有的 Web App 部署。

3 具體應用範圍 (AI 能為您做什麼)

應用類型 具體應用場景 GAS 核心責任
🤖 聊天機器人 LINE / Discord 靈感收藏盒、自動回應客服系統。 作為 Webhook 接收端，將對話紀錄寫入 Google 試算表。
📊 自動化點名與登錄 班級點名工具、學生成績登錄、課堂實作表單。 用試算表當資料庫，提供前端網頁快速讀寫與統計。
⏰ 定時排程與爬蟲 每天早上定時爬取天氣、新聞，並發送 LINE Notify。 使用 GAS 內建的排程觸發器（Trigger）定時執行爬取與發送。
📧 辦公室報表與通知 依據名單自動發送 Gmail 證書信、自動生成 Google 日曆行程。 串接 Google 內建服務（MailApp、CalendarApp）一鍵批次處理。

4 ⚠️ 防呆與安全守則

🔒 保持 Web App 網址不變 (重要)

更新程式碼時，請命令 AI 使用現有的部署 ID 進行更新：
npx clasp deploy -i [現有的部署ID] -d "版本描述"
切勿執行不帶參數的 clasp deploy，因為那會建立一個「全新的網址」，導致您所有綁定 LINE 或前端的 Webhook 網址全部失效！
