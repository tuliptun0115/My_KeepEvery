# 💡 AI 協作 Google Apps Script (GAS) clasp 使用指南

本指南是專為與 AI 協作開發 Google Apps Script (GAS) 設計的**一鍵部署懶人包**。透過 Google 官方的 `clasp` 工具，您再也不需要手動複製貼上程式碼，AI 就可以直接在本地幫您把程式碼同步並更新到雲端！

---

## 🌟 為什麼要用 clasp？
* **拒絕複製貼上**：程式碼寫完，AI 只要下一行指令就直接同步到雲端。
* **網址不變更新**：更新程式碼時，可以自動覆蓋既有版本，**維持既有的 Web App 網址不變**，避免 LINE Bot 或前端連結失效。
* **安全可靠**：完全使用 Google 官方的 OAuth 安全機制授權。

---

## 🛠️ 第一步：一生一次的前置準備

在使用之前，請確保您的帳號已開啟 API 存取權限：
1. 瀏覽 **[Google Apps Script 使用者設定](https://script.google.com/home/usersettings)**。
2. 找到 **Google Apps Script API**，將其切換為 **「開啟 (ON)」**。
*(這步如果不開，本地指令會被拒絕存取。)*

---

## 🏃‍♂️ 第二步：三步自動化操作步驟 (照著做就行)

當您請 AI 修改或新增了 GAS 程式碼後，照著以下步驟執行：

### 1. 登入授權 (AI 啟動，您在瀏覽器確認)
在 AI 對話中，AI 會執行 `clasp login` 並提供您一個登入網址：
1. 點擊 AI 給您的**授權連結**。
2. 登入您的 Google 帳號，並點選「允許」授權。
3. 網頁會自動跳轉並顯示 `Logged in! You may close this page.`，此時即可關閉網頁。

### 2. 獲取並提供 Script ID (僅在初次連結專案時需要)
請提供您要部署的 GAS 專案 **Script ID** 給 AI：
* **如何找到 Script ID**：
  打開您的 Apps Script 專案網頁 ➡️ 點選左側選單的 **「專案設定 ⚙️」** (齒輪圖示) ➡️ 複製裡面那一長串的 **「Script ID」** 給 AI。
* **AI 會為您做什麼**：
  AI 會自動在本地建立 `.clasp.json` 設定檔，並建立一個獨立的 `gas/` 目錄，把雲端的 Manifest 檔 (`appsscript.json`) 拉取下來，以防 Next.js 專案的其他無效檔案被誤推上雲端。

### 3. 一鍵推送到雲端與發布
AI 將會為您自動執行：
1. **`clasp push`**：將本地最新程式碼同步到您的雲端專案中。
2. **`clasp deploy -i <部署ID>`**：直接覆蓋您現有的部署，保持 Web App 網址不變！

## 🎯 clasp 的應用範圍

只要需要將 Google 試算表（Google Sheets）或雲端硬碟（Google Drive）當作資料庫、自動化工具的場景，clasp 都能極大提升開發速度：

| 應用場景 | 說明 |
|---|---|
| **LINE Bot 後端** | 自動接收 LINE 訊息並將資料存入 Google 試算表。 |
| **自動化表單系統** | 例如飲料訂單、學生成績登錄、班級點名系統，資料直接寫入並整理。 |
| **Gmail / 日曆自動化** | 依據試算表名單，自動定時排程發送 E-mail 或在 Google 日曆建立行程。 |
| **定時排程通知** | 每天早上自動爬取氣象或最新消息，並發送到 LINE 群組。 |

## ⚠️ 常見問題與防呆提醒

### Q1：為什麼點擊授權連結後，瀏覽器顯示「無法連上這個網站（ERR_CONNECTION_REFUSED）」？
* **原因**：因為本地的登入進程尚未啟動，或是被系統重啟中斷了，導致沒有程式在監聽本機 Port。
* **解決做法**：請直接請 AI 重新啟動 `clasp login`，AI 會為您提供一個**全新的授權網址**，重新點擊即可。

### Q2：使用 clasp deploy 會導致我的 LINE 機器人失效嗎？
* **不會**，前提是必須使用 `clasp deploy -i <舊部署ID>` 來更新。AI 會自動從環境變數讀取您的舊部署 ID 進行安全更新。**千萬不要執行無參數的 `clasp deploy`**，因為那會建立一個全新的網址。
